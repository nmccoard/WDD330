    Chapter 11 - Further Functions
  • Memorizing or caching can be used if a function takes some time to compute a return value, we can save the result in a cache property. Then if the same argument is used again later, we can return the value from the cache, rather than having to compute the result again.
  • Immediately Invoked Function Expression (IIFE) are anonymous self invoking functions that run right after they have been defined.
  • IIFEs are a useful way of performing a task while keeping any variables wrapped up within the scope of the function.
  • IIFEs can be used to create Temporary Variables (see example)
  • JavaScript is single threaded, where only one task can happen at a time
  • A promise represents the future result of an asynchronous operation.
  • An “inner” function, which is declared inside another function, has full access to all of the variables declared inside the scope of the function in which it’s declared (the “outer” function).
  • Closures is kind of like global variables?
  • Roles for a pure function:
    1) The return value of a pure function should only depend on the values provided as arguments. It doesn’t rely on values from somewhere else in the program.
    2) There are no side-effects. A pure function doesn't change any value or data elsewhere in the program. It only make non-destructive data transformations and returns new values.
    3) Referential transparency. Given the same arguments, a pure function will always return the same result.
