    • Chapter 12 - Object-Oriented Programming in JS
  • Three main concepts: Encapsulation, Polymorphism, and Inheritance
  • Encapsulation: is keeping the inner workings hidden by putting all the programming logic inside an object and making methods available to implement the functionality.
  • Polymorphism: is the ability to use the same process can be used by different objects. With various objects being able to sharing the same method, but also letting them have the ability to override shared methods with a more specific implementation when needed.
  • Inheritance: is being able to take features form one object and then adding more properties and/or methods to it to make a new object.
  • The class declaration syntax works in exactly the same way as the constructor function syntax, because it’s actually just syntactic sugar that is implemented in the same way in the background.
  • The prototype can be used to add any new properties and methods after the class has been declared. It should be used to define any properties that will remain the same for every instance of the class.

  • Model-View-Controller (MVC) separates an application into three distinct, independent components that interact with each other
  • Models are objects that implement the functionality for creating, reading, updating and deleting (known as CRUD tasks) specific pieces of information about the application,
  • Views provide a visual representation of the model showing all the relevant information. In a web application, this would be the HTML displayed on a web page.
  • Controllers link models and views together by communicating between them. They respond to events, which are usually inputs from a user, process the information, and then update the model and view accordingly.
  • React and Vue.js both use the concept of a virtual DOM to speed up the process of updating HTML pages
  • As a general rule, you should aim to use well-commented and descriptive code in development and minified code in production (since there’s no need for end users to read comments).
