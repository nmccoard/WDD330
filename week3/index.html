    Object methods, "this"
  • Object-oriented programming or OPP is when we write our code using objects to represent entities.
  • A function that is the property of an object is called its method.
  • The value of this is the object “before dot”, the one used to call the method.
  • Arrow functions are special: they don’t have their “own” this. They use the outer normal function's this
  • The value of this is defined at run-time.

    • This is JavaScript
  • When this is called outside of any function, in a global context, this defaults to the window object in the browser.
  • this within any method refers to the object itself.
  • this is always set to window in a simple function
  • this in an arrow function is always the same as this around it (in its immediate scope).
  • this in event listeners is set to the element that fired it (ie the button)

    • Chapter 5 - "Objects"
  • Object literals - allow objects to be created quickly without the need for defining a class as you would in other languages.
  • Object literals that is created directly in the code by wrapping all its properties/methods in {}.
  • Object literals use key-value pairs, separated by commas.
  • Object literals can have computed property keys. This is done by using JavaScript inside [].
  • Symbol data type can be used as a computed property key. You can access the property using the [] notation
  • Each symbol has a unique value, which means that using them as property keys avoids any naming clashes if you mistakenly use the same value for two different property keys
  • Object properties do not always appear in the same order or the order they were entered. Never rely on a property to be in a certain oder.
  • Objects can be nested (ie objects inside of objects
  • Objects are assigned by reference (pointer a spot in memory). Any changes made using the either reference will affect the same object.
  • When using stringify() to turn objects in to a JSON formatted string it will ignore methods and not include them.
